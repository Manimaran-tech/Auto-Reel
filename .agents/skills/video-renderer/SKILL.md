# AutoReel Video Renderer Skill

## Overview
This skill teaches you how to build a programmatic video editing engine using Python and MoviePy that produces Instagram Reel quality vertical videos from static product images and AI-generated voiceovers.

## Core Principles
1. **Output format:** Always render at 1080x1920 (9:16 vertical ratio) for Instagram Reels.
2. **No GPU:** All rendering must work on CPU-only hardware (i5 + 16GB RAM).
3. **Target length:** 15-30 seconds per video.
4. **File size:** Keep the output MP4 under 50MB (Instagram's upload limit for Reels).

## Video Assembly Pipeline

### Step 1: Image Preparation
- Load product images using `Pillow`.
- Resize each image to fill a 1080x1920 canvas:
  - If the image is landscape, create a blurred version as background, overlay the sharp image centered.
  - If the image is portrait or square, center-crop to fill.
- Apply a subtle color grade for visual consistency.

### Step 2: Ken Burns Effect
- For each image, create a `moviepy.video.VideoClip` that slowly zooms from 1.0x to 1.15x scale.
- Alternate directions: zoom-in on odd images, zoom-out on even images.
- This transforms static images into cinematic motion.

```python
from moviepy import ImageClip
import numpy as np

def make_ken_burns_clip(image_path, duration, zoom_in=True):
    """Create a Ken Burns zoom effect on a static image."""
    clip = ImageClip(image_path).with_duration(duration)
    w, h = clip.size

    def zoom_effect(get_frame, t):
        # Calculate zoom factor based on time
        if zoom_in:
            zoom = 1 + (0.15 * t / duration)  # 1.0 → 1.15
        else:
            zoom = 1.15 - (0.15 * t / duration)  # 1.15 → 1.0

        frame = get_frame(t)
        # Calculate crop dimensions
        new_w, new_h = int(w / zoom), int(h / zoom)
        x = (w - new_w) // 2
        y = (h - new_h) // 2
        cropped = frame[y:y+new_h, x:x+new_w]

        from PIL import Image
        resized = np.array(Image.fromarray(cropped).resize((w, h), Image.LANCZOS))
        return resized

    return clip.transform(zoom_effect)
```

### Step 3: Audio Integration
- Load `voiceover.mp3` using MoviePy's `AudioFileClip`.
- Calculate total video duration from audio duration.
- Divide duration equally among product images.
- Optionally, mix background music at 15% volume using `CompositeAudioClip`.

### Step 4: Subtitle Rendering
- Parse the `.vtt` file generated by `edge-tts` to get word-level timestamps.
- Group words into chunks of 3-5 words each.
- For each chunk, create a `TextClip`:
  - Font: Bold sans-serif (Montserrat Bold, Arial Black, or Impact).
  - Size: 56-64px.
  - Color: White with black stroke/outline.
  - Position: Center-bottom of the frame (70% down from top).
  - Each chunk appears only during its timestamp window.
- Overlay all TextClips onto the video using `CompositeVideoClip`.

## Guidelines
- Always check that FFmpeg is installed before attempting to render.
- Use `preset="ultrafast"` during development for faster iteration, switch to `"medium"` for final output.
- Keep the total number of images between 3-5 for a 15-second reel.
- If rendering takes too long, reduce resolution to 720x1280 for testing.

## Common MoviePy Pitfalls
- MoviePy v2.x has breaking API changes from v1.x. Use `moviepy` v2.1+ syntax.
- In MoviePy v2: use `clip.with_duration()` instead of `clip.set_duration()`.
- In MoviePy v2: use `clip.with_position()` instead of `clip.set_position()`.
- In MoviePy v2: use `clip.with_start()` instead of `clip.set_start()`.
- Always close clips after use to avoid memory leaks.
- Use `clip.close()` or context managers.
